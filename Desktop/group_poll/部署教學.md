# Group Meeting 時段調查 App — 部署教學（白話版）

整個流程分四步：**建試算表 → 開 Google API 拿金鑰 → 設定 Streamlit Secrets → 設定 Slack**。照著做即可。

---

## 步驟 1：建立 Google 試算表

1. 到 Google Drive 新建一份試算表，檔名命名為 **`GroupMeetingPoll`**（要和 `app.py` 裡的 `GSHEET_NAME` 一致）。
2. 程式第一次執行時會自動建立 `votes` 分頁與表頭，你不用手動加欄位。
3. 這份試算表稍後要「分享」給機器人帳號（見步驟 2 最後）。

---

## 步驟 2：去 Google Cloud Console 開啟什麼 API？

到 <https://console.cloud.google.com/>，建立一個新專案（或用現有的），然後：

### 2-1. 要開啟的 API（兩個）

進入「**API 與服務 → 程式庫**」，搜尋並啟用這兩個：

- **Google Sheets API** ← 讀寫試算表用
- **Google Drive API** ← gspread 用檔名開檔需要它

> 白話：Sheets API 負責「改表格內容」，Drive API 負責「用名字找到那份檔案」。兩個都要開。

### 2-2. 建立 Service Account（機器人帳號）並下載 JSON 金鑰

1. 「**API 與服務 → 憑證 → 建立憑證 → 服務帳戶（Service Account）**」。
2. 取個名字（例如 `poll-bot`），建立完成。
3. 點進該服務帳戶 → **金鑰（KEYS）→ 新增金鑰 → 建立新的金鑰 → 選 JSON** → 下載。
4. 你會拿到一個 `.json` 檔，裡面長這樣（重點欄位）：

```json
{
  "type": "service_account",
  "project_id": "your-project-id",
  "private_key_id": "xxxxxxxx",
  "private_key": "-----BEGIN PRIVATE KEY-----\n....\n-----END PRIVATE KEY-----\n",
  "client_email": "poll-bot@your-project-id.iam.gserviceaccount.com",
  "client_id": "1234567890",
  "auth_uri": "https://accounts.google.com/o/oauth2/auth",
  "token_uri": "https://oauth2.googleapis.com/token",
  "auth_provider_x509_cert_url": "https://www.googleapis.com/oauth2/v1/certs",
  "client_x509_cert_url": "https://www.googleapis.com/robot/v1/metadata/x509/..."
}
```

### 2-3. ⚠️ 最關鍵一步：把試算表分享給機器人

複製 JSON 裡的 **`client_email`**（像 `poll-bot@xxx.iam.gserviceaccount.com`），
回到步驟 1 的 Google 試算表 → 右上角「**共用 / Share**」→ 貼上這個 email →
權限設為 **編輯者（Editor）** → 傳送。

> 忘記這步的話，程式會出現 `SpreadsheetNotFound` 或權限錯誤。99% 的問題都出在這裡。

---

## 步驟 3：Streamlit Secrets 要怎麼填（TOML 格式）

Streamlit 的 Secrets 是 **TOML 格式**，不是直接貼 JSON。你要把剛剛的 JSON「翻譯」成下面這樣。

部署到 Streamlit Community Cloud 後：**App 右下角 ⋮ → Settings → Secrets**，貼上以下內容：

```toml
# ============ Google Service Account ============
[gcp_service_account]
type = "service_account"
project_id = "your-project-id"
private_key_id = "xxxxxxxx"
private_key = "-----BEGIN PRIVATE KEY-----\nMIIE...省略...\n-----END PRIVATE KEY-----\n"
client_email = "poll-bot@your-project-id.iam.gserviceaccount.com"
client_id = "1234567890"
auth_uri = "https://accounts.google.com/o/oauth2/auth"
token_uri = "https://oauth2.googleapis.com/token"
auth_provider_x509_cert_url = "https://www.googleapis.com/oauth2/v1/certs"
client_x509_cert_url = "https://www.googleapis.com/robot/v1/metadata/x509/poll-bot%40your-project-id.iam.gserviceaccount.com"

# ============ Slack Webhook ============
[slack]
webhook_url = "https://hooks.slack.com/services/XXX/YYY/ZZZ"
```

### 三個最容易踩的雷

1. **`private_key` 的換行**：JSON 裡是 `\n`，貼進 TOML 時請「保留 `\n` 字串」，用**雙引號**包起來（如上）。不要真的去按 Enter 換行。
2. **每個值都要用引號**：TOML 字串一律加雙引號 `"..."`。
3. **section 名稱固定**：一定要是 `[gcp_service_account]` 和 `[slack]`，程式靠這兩個名字讀取（`st.secrets["gcp_service_account"]`、`st.secrets["slack"]`）。

> 本機測試時：在專案目錄建立 `.streamlit/secrets.toml`，貼一樣的內容即可。**記得把它加進 `.gitignore`，千萬別上傳到 GitHub。**

---

## 步驟 4：設定 Slack Incoming Webhook

1. 到 <https://api.slack.com/apps> → **Create New App → From scratch**，選擇要發訊息的 workspace。
2. 左側 **Incoming Webhooks → 開啟 Activate**。
3. **Add New Webhook to Workspace** → 選一個頻道 → 授權。
4. 複製產生的 Webhook URL（`https://hooks.slack.com/services/...`），貼到上面 Secrets 的 `slack.webhook_url`。

> 訊息中的 `<!channel>` 會觸發 @channel 通知整個頻道。

---

## 步驟 5：部署到 Streamlit Community Cloud

1. 把 `app.py` 和 `requirements.txt` 推上 GitHub repo（**不要**上傳金鑰）。
2. 到 <https://share.streamlit.io/> → **New app** → 選 repo、branch、主程式 `app.py`。
3. 部署後在 **Settings → Secrets** 貼上步驟 3 的 TOML，存檔，App 會自動重啟。
4. 完成！把網址貼到 Slack 給大家投票。

## 日常使用小抄

- **改候選時段**：編輯 `app.py` 最上方的 `ALL_SLOTS`。
- **排除某時段**（例如必修課）：把該時段的 label 字串加進 `EXCLUDE_SLOTS`，它就不會出現在投票選項。
- **調整權重**：改 `ROLE_WEIGHT`（目前碩博 2 分、專題生 1 分）。
- **覆蓋投票**：同一姓名重新送出即覆蓋舊紀錄，不會重複計票。
- **計分邏輯**：田老師沒勾 → 該時段 −1（老師不行）；田老師有勾 → 分數 = 2×碩博人數 + 1×專題生人數。

### 熱力圖中文變方框？（已內建雙保險解法）

如果你看到圖表的中文還是方框（豆腐字），**99% 是因為你還在跑舊版 `app.py`**——
舊版圖表標題是「Group Meeting …-1 = …」且是紅黃配色；新版是綠色卡片、標題為
「Group Meeting 時段投票熱力圖」。請務必用新版檔案覆蓋並 **Reboot** App。

本專案用兩道保險確保中文一定顯示：

1. **內建字型檔（最可靠）**：repo 內附 `fonts/NotoSansCJK-Regular.ttc`，
   程式 `setup_chinese_font()` 會優先載入它。**不管部署到哪都有中文**。
2. **packages.txt**：內含 `fonts-noto-cjk`，Streamlit Cloud 會額外安裝系統字型備援。

> ⚠️ 上傳 GitHub 時，**`fonts/` 資料夾連同裡面的 .ttc 一定要一起上傳**（檔案約 19MB，
> 在 GitHub 100MB 上限內，沒問題）。本機（Windows/macOS）即使沒這資料夾也會自動用系統字型。

### 你的 repo 應該長這樣

```
你的專案/
├── app.py
├── requirements.txt
├── packages.txt
└── fonts/
    └── NotoSansCJK-Regular.ttc
```

### 改了 code 圖卻沒變？（清快取）

Streamlit Cloud：右下角 ⋮ →「**Reboot app**」。
本機：終端機按 Ctrl+C 停掉再 `streamlit run app.py`，或網頁右上角選「Clear cache」後 Rerun。
